45  /// <summary>
46  /// <para>Empty the output buffer --- called whenever buffer fills up.</para>
47  /// <para>
48  /// In typical applications, this should write the entire output buffer
49  /// (ignoring the current state of next_output_byte and free_in_buffer),
50  /// reset the pointer and count to the start of the buffer, and return true
51  /// indicating that the buffer has been dumped.
52  /// </para>
53  /// <para>
54  /// In applications that need to be able to suspend compression due to output
55  /// overrun, a false return indicates that the buffer cannot be emptied now.
56  /// In this situation, the compressor will return to its caller (possibly with
57  /// an indication that it has not accepted all the supplied scanlines). The
58  /// application should resume compression after it has made more room in the
59  /// output buffer. Note that there are substantial restrictions on the use of
60  /// suspension --- see the documentation.
61  /// </para>
62  /// <para>
63  /// When suspending, the compressor will back up to a convenient restart point
64  /// (typically the start of the current MCU). next_output_byte and free_in_buffer
65  /// indicate where the restart point will be if the current call returns false.
66  /// Data beyond this point will be regenerated after resumption, so do not
67  /// write it out when emptying the buffer externally.
